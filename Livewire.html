Lenguajes: Livewire

Inicio rápido

Instalar Livewire

Incluir el PHP.

        
1composer require livewire/livewire

Incluye el JavaScript (en cada página que use Livewire).

        
1...
2    @livewireStyles
3</head>
4<body>
5    ...
6 
7    @livewireScripts
8</body>
9</html>

Crear un componente

Ejecuta el siguiente comando para generar un nuevo componente Livewire llamado counter.

        
1php artisan make:livewire counter

Ejecutar este comando generará la siguiente pareja de archivos. Uno es el controlador y el otro la vista:

            
 1namespace App\Http\Livewire;
 2 
 3use Livewire\Component;
 4 
 5class Counter extends Component
 6{
 7    public function render()
 8    {
 9        return view('livewire.counter');
10    }
11}
        
            
1<div>
2    ...
3</div>

Agreguemos algo de texto a la vista para que podamos ver algo tangible en el navegador.

La vista de un componente Livewire debe tener un único elemento div raiz
            
1<div>
2    <h1>Hello World!</h1>
3</div>

Incluir el componente

Piensa en los componentes Livewire como en los includes de Blade. Se puede insertar <livewire:some-component /> en cualquier lugar de una vista Blade y se renderizará.

        
 1<head>
 2    ...
 3    @livewireStyles
 4</head>
 5<body>
 6    <livewire:counter /> 
 7 
 8    ...
 9 
10    @livewireScripts
11</body>
12</html>

Visualizarlo en el navegador

En el navegador, carga la página en la que has incluido Livewire. Deberías ver "¡Hola Mundo!".

Añadir la funcionalidad "contador"

Sustituir el contenido generado de la clase y la vista del componente contador por lo siguiente:

            
 1class Counter extends Component
 2{
 3    public $count = 0;
 4 
 5    public function increment()
 6    {
 7        $this->count++;
 8    }
 9 
10    public function render()
11    {
12        return view('livewire.counter');
13    }
14}
app/Http/Livewire/Counter.php 
            
1<div style="text-align: center">
2    <button wire:click="increment">+</button>
3    <h1>{{ $count }}</h1>
4</div>

Verlo en el navegador

Ahora recarga la página en el navegador, deberías ver el componente contador renderizado. Si haces clic en el botón "+", la página se actualizará automáticamente sin necesidad de recargarla. Magia 🧙.️

En general, una cosa tan trivial como este "contador" es más adecuada para algo como AlpineJS, sin embargo es una de las mejores maneras de entender fácilmente el funcionamiento de Livewire.

FUNDAMENTOS

Instalación

Requisitos

  1. PHP 7.2.5 o superior
  2. Laravel 7.0 o superior

Consulta el archivo composer.json en Github para ver la lista completa de requisitos de los paquetes.

Instalación del paquete

        
1composer require livewire/livewire

Incluir los activos

Añadir las siguientes directivas Blade en la etiqueta head, y antes de la etiqueta body al final de la plantilla.

        
 1<html>
 2<head>
 3    ...
 4    @livewireStyles
 5</head>
 6<body>
 7    ...
 8    @livewireScripts
 9</body>
10</html>

También es posible utilizar la sintaxis de etiquetas.

        
1<livewire:styles />
2...
3<livewire:scripts />

Ya está. Es todo lo que necesitas para empezar a usar Livewire. El resto de la página es opcional.

Publicación del archivo de configuración

Livewire tiene como objetivo "cero-configuración" inicial, pero algunos usuarios requieren más opciones de configuración.

Se puede publicar el archivo de configuración de Livewire con el siguiente comando artisan:

        
1php artisan livewire:publish --config

Publicación de activos del frontend

Si prefieres que los activos JavaScript sean provistos por tu servidor web y no a través de Laravel, utiliza el comando livewire:publish:

        
1php artisan livewire:publish --assets

Para mantener los activos actualizados y evitar problemas en futuras actualizaciones, recomendamos encarecidamente añadir este comando a los scripts post-autoload-dump del archivo composer.json:

        
1{
2    "scripts": {
3        "post-autoload-dump": [
4            "Illuminate\\Foundation\\ComposerScripts::postAutoloadDump",
5            "@php artisan package:discover --ansi",
6            "@php artisan vendor:publish --force --tag=livewire:assets --ansi"
7        ]
8    }
9}

Configuración de la URL de la base de activos

Por defecto, Livewire sirve su parte JavaScript (livewire.js) desde la siguiente ruta en la aplicación: /livewire/livewire.js.

La etiqueta script que se genera por defecto es:

<script src="/livewire/livewire.js">

Hay dos escenarios que causarán que este comportamiento por defecto se rompa:

  1. Si publicas los recursos de Livewire y ahora los estás sirviendo desde una subcarpeta como "assets".
  2. La aplicación está alojada en una ruta no raíz de su dominio. Por ejemplo: https://your-laravel-app.com/application. En este caso, los recursos reales se servirán desde /application/livewire/livewire.js, pero la etiqueta de script generada intentará obtener /livewire/livewire.js.

Para resolver cualquiera de estos problemas, se puede configurar "asset_url" en config/livewire.php para personalizar lo que se añade al atributo src="".

Por ejemplo, después de publicar el archivo de configuración de Livewire, estos son los ajustes que solucionarían los dos problemas anteriores:

  1. 'asset_url' => '/assets'
  2. 'asset_url' => '/aplicación'

Creación de componentes

Introducción

Ejecuta el siguiente comando artisan para crear un nuevo componente Livewire:

        
1php artisan make:livewire ShowPosts

Livewire también admite la notación "kebab" para los componentes.

        
1php artisan make:livewire show-posts

Se han creado dos nuevos archivos en el proyecto:

app/Http/Livewire/ShowPosts.php

resources/views/livewire/show-posts.blade.php

Si quieres crear componentes dentro de subcarpetas, puedes recurrir a la siguientes sintaxis:

        
1php artisan make:livewire Post\\Show
2php artisan make:livewire Post/Show
3php artisan make:livewire post.show

Ahora, los dos archivos creados estarán en subcarpetas:

app/Http/Livewire/Post/Show.php

resources/views/livewire/post/show.blade.php

Generación de pruebas

Opcionalmente, se puede incluir el indicador --test al crear un componente, y también se creará un archivo de prueba.

        
1php artisan make:livewire ShowPosts --test

Componentes inline

Si quieres crear componentes Inline (componentes sin archivos .blade.php), puedes añadir la bandera --inline al comando:

        
1php artisan make:livewire ShowPosts --inline

De este modo, sólo se creará un archivo

app/Http/Livewire/ShowPosts.php

Así es como quedaría:

        
1class ShowPosts extends Component
2{
3    public function render()
4    {
5        return <<<'blade'
6            <div></div>
7        blade;
8    }
9}
